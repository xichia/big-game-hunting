# Big Game Hunting Design Gates

Use these gates before any major mechanic, direction, or build task advances.

## Gate 1: Simplicity Gate

A candidate passes if:

- A new player can understand the basic objective quickly.
- A first turn or first action is obvious enough to attempt.
- Rules burden is proportional to payoff.
- The game can be explained without a long dependency chain.

Review questions:

```text
Can a beginner make a reasonable move?
Can they recover from early mistakes?
Can they enjoy the game before mastering it?
```

## Gate 2: Mastery Gate

A candidate passes if:

- Experienced players can improve over time.
- There are meaningful reads, counters, timing decisions, or tradeoffs.
- Strategies vary by context.
- There is no obvious solved path.

Review questions:

```text
What can experts get better at?
What separates novice play from skilled play?
What would high-level counterplay look like?
```

## Gate 3: Replayability Gate

A candidate passes if:

- Repeated plays produce different situations.
- Multiple strategies remain viable.
- Players can tell different stories after different games.
- The game does not collapse once the obvious strategy is known.

## Gate 4: Social Appeal Gate

A candidate passes if:

- It is fun to teach.
- It is fun to watch.
- It creates memorable moments.
- Players can describe what happened afterward.
- It supports social spread.

## Gate 5: Production Gate

A candidate passes if:

- It can be prototyped cheaply.
- The smallest meaningful test is clear.
- Its complexity is feasible for the current build stage.
- It does not require a full product before proving its core promise.

## Gate Result Labels

```text
GREEN — ready to advance
YELLOW — promising, revise or test further
RED — do not advance in current form
VETO — contradicts source-of-truth principles
QUESTION — needs human judgment
```
